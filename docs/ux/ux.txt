Киллер-фича: Signature Chain с hash reference
На основе бэкенда с hash-chain в audit_log (internal/db/migrations/043_tenant_quotas.up.sql), каждая TO должна включать hash предыдущего TO этого же устройства:
TO TO-201 hash: SHA256(TO-200 hash + TO-201 data)
Это означает, что TO TO-201 становится криптографически связанным с TO TO-200 → невозможно подделать историю ТО. Это главный selling point для B2G-тендеров с КИИ-статусом (187-ФЗ).

  GET    /api/v1/sites/tree?depth=5           → Hierarchical view
  GET    /api/v1/devices/{id}/tunnel          → WebSocket endpoint для SSH tunnel
  GET    /api/v1/devices/{id}/screenshots     → Last N screenshots
  GET    /api/v1/devices/{id}/raw-telemetry   → Unprocessed metrics
  POST   /api/v1/devices/onboard-batch       → Bulk import + auto-assign

QR:
  POST   /api/v1/qr/generate-batch            → Bulk PDF с QR
