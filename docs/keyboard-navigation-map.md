## 2. Focus order по страницам

### 2.1 Devices (`/devices`)

```
1. Skip link → "Перейти к содержимому"
2. Header: логотип → Workspace switcher → язык
3. Основная панель: "Smart Onboarding" → "Add Device" → "Filter"
4. Строка поиска → фильтры (Site, Status)
5. Таблица устройств (по строкам, затем по ячейкам слева направо)
6. Кнопки действий в каждой строке (Edit → Delete)
7. Пагинация (если есть)
8. Sidebar навигация
```

### 2.2 Work Orders (`/work-orders`)

```
1. Skip link
2. Header
3. "Create Work Order" → фильтры
4. Kanban-доска / таблица (карточки слева направо, сверху вниз)
5. Sidebar
```

### 2.3 Dashboard (`/dashboard`)

```
1. Skip link
2. Header
3. Виджеты: Stats → Charts → Alerts (слева направо, сверху вниз)
4. Sidebar
```

### 2.4 Device Detail (`/devices/:id`)

```
1. Skip link
2. Header
3. Device info → вкладки (Overview, Alarms, Logs, RCA)
4. Содержимое активной вкладки
5. Кнопки действий
6. Sidebar
```

### 2.5 Modal / Wizard

```
1. Заголовок модалки (автофокус)
2. Кнопка закрытия (X)
3. Содержимое модалки (слева направо, сверху вниз)
4. Footer-кнопки (Cancel → Back → Next/Create)
5. Focus trap: Tab циклит внутри модалки
```

## 3. Skip link

Каждая страница содержит скрытую ссылку **"Перейти к содержимому"** (`useSkipLink` из [`useAccessibility.ts`](frontend/src/hooks/useAccessibility.ts:10)).

- Появляется при первом нажатии `Tab` на странице
- Ведёт к `#main-content`
- После перехода фокус устанавливается на контент, tabindex временно снимается

```html
<a href="#main-content" class="skip-link" ref={skipLinkRef} onClick={handleSkip}>
  Перейти к содержимому
</a>
```

## 4. Focus trap (модальные окна)

Все модальные окна используют `useFocusTrap` из [`useAccessibility.ts`](frontend/src/hooks/useAccessibility.ts:10).

Поведение:
- При открытии — фокус на первый фокусируемый элемент внутри модалки
- `Tab` циклит только внутри модалки
- `Shift+Tab` — обратный цикл
- `Escape` — закрывает модалку
- При закрытии — фокус возвращается на элемент, открывший модалку

---

## 5. Тестирование

1. Откройте любую страницу
2. Нажмите `Tab` — должен появиться skip link
3. Пройдите `Tab` по всем элементам — фокус должен следовать порядку из секции 2
4. Откройте модальное окно — `Tab` не должен выходить за его пределы
5. Нажмите `?` — должен открыться Cheatsheet
6. Проверьте все шорткаты из таблицы

---

## 6. Известные ограничения

- Графики (Chart.js/Recharts) — фокус только на контейнере, не на точках данных
- Drag-and-drop виджеты — клавиатурное управление реализовано частично
- Кастомные селекты — используют `aria-activedescendant` для навигации
